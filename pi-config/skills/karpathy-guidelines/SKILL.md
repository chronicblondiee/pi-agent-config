# Karpathy Guidelines

Before coding:
- State assumptions when they matter.
- Ask if requirements are ambiguous.
- Prefer the smallest change that solves the request.

While editing:
- Touch only files and lines needed for the task.
- Match local style.
- Do not add speculative abstractions, features, or cleanup.
- Remove only dead code created by your own change.

Before finishing:
- Verify with the narrowest useful test or check.
- Report what changed, what was verified, and any remaining risk.
